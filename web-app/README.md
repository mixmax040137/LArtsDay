# ShoeTracker (Web)

เวอร์ชันเว็บของ ShoeTracker — เปิดผ่านเบราว์เซอร์ได้ทั้งบนเดสก์ท็อปและมือถือ (responsive)
ใช้งานได้ทันทีโดยไม่ต้องติดตั้งอะไรเพิ่ม เพราะเป็น static HTML/CSS/JS ล้วนๆ (เหมือน `index.html` ของแบบสอบถามในรีโปนี้)

## เปิดใช้งาน

เปิดไฟล์ `web-app/index.html` ในเบราว์เซอร์ได้เลย หรือโฮสต์ผ่าน GitHub Pages/Netlify เหมือนที่ทำกับหน้าแบบสอบถามอยู่แล้ว
(แนะนำให้โฮสต์ออนไลน์หากต้องการใช้ฟีเจอร์เชื่อมต่อ Strava เพราะต้องมี URL จริงสำหรับ OAuth redirect)

## ฟีเจอร์

- เพิ่ม/แก้ไข/ทำเครื่องหมายเลิกใช้/ลบรองเท้า พร้อมยอดระยะทางสะสม
- บันทึกการวิ่งด้วยมือ: วันที่ เวลา ระยะทาง ระยะเวลา หมายเหตุ
- **หน้าจอปรับตามขนาดจอ**: บนจอกว้าง (≥860px) แสดงรายชื่อรองเท้าและรายละเอียดคู่กัน (master–detail)
  บนมือถือแสดงทีละหน้าจอพร้อมปุ่มย้อนกลับ
- นำเข้าการวิ่งจากไฟล์ CSV (เช่น export มาจากแอพอื่น หรือจาก Apple Health ผ่านเครื่องมือ export)
- เชื่อมต่อ Strava (ไม่บังคับ, ดูวิธีตั้งค่าด้านล่าง): ดึงรายชื่อ Gear และกิจกรรมวิ่งมาให้อัตโนมัติ
- ข้อมูลทั้งหมดเก็บใน `localStorage` ของเบราว์เซอร์ — ไม่มีบัญชีผู้ใช้ ไม่มีฐานข้อมูลกลาง (ยังไม่ซิงค์ข้ามอุปกรณ์/เบราว์เซอร์)

**Apple Health / Apple Fitness ใช้ไม่ได้จากเว็บ** — Apple ไม่เปิด HealthKit ให้เว็บเบราว์เซอร์เข้าถึง (เป็นข้อจำกัดของ Apple เอง ไม่ใช่ของแอพนี้)
เชื่อมต่อ Health ได้เฉพาะในแอพ iOS ที่อยู่ในโฟลเดอร์ `ios-app/` เท่านั้น สำหรับเว็บให้ export ข้อมูลจากที่อื่นเป็น CSV แล้วนำเข้าแทน

## การเชื่อมต่อ Strava (ไม่บังคับ)

เว็บไม่สามารถฝัง Strava Client Secret ไว้ในหน้าเว็บได้อย่างปลอดภัย (ใครก็ดูซอร์สโค้ดเห็นได้) จึงต้องมีตัวกลาง (proxy)
เก็บ secret ไว้ฝั่งเซิร์ฟเวอร์แทน — ใช้ **Google Apps Script** แบบเดียวกับที่แบบสอบถามในรีโปนี้ใช้อยู่แล้ว

### 1) สมัครแอพกับ Strava
ไปที่ https://www.strava.com/settings/api สร้างแอพใหม่ แล้วคัดลอก **Client ID** และ **Client Secret**

ตั้งค่า **Authorization Callback Domain** เป็นโดเมนที่คุณจะโฮสต์หน้านี้ (เช่น `ชื่อผู้ใช้.github.io` ถ้าใช้ GitHub Pages)

### 2) Deploy ตัวกลาง (Apps Script)
1. ไปที่ https://script.google.com → **New project**
2. ลบโค้ดเดิมทั้งหมด แล้ววางเนื้อหาจาก [`apps-script/StravaProxy.gs`](apps-script/StravaProxy.gs)
3. เมนู **Project Settings → Script Properties** เพิ่ม 2 ค่า:
   - `STRAVA_CLIENT_ID`
   - `STRAVA_CLIENT_SECRET`
4. **Deploy → New deployment** → ประเภท Web app
   - Execute as: **Me**
   - Who has access: **Anyone**
5. คัดลอก Web app URL (ลงท้าย `/exec`)

### 3) ตั้งค่าในหน้าเว็บ
เปิดไฟล์ [`app.js`](app.js) แก้ค่า `CONFIG` ด้านบนไฟล์:
```js
const CONFIG = {
  STRAVA_CLIENT_ID: "วาง Client ID จาก Strava",
  STRAVA_PROXY_ENDPOINT: "วาง Web app URL จากขั้นตอนที่แล้ว",
};
```

### 4) ใช้งาน
ไปที่แท็บ **ตั้งค่า** ในหน้าเว็บ กด "เชื่อมต่อ Strava" ระบบจะพาไปหน้า Strava เพื่อล็อกอิน/อนุญาตสิทธิ์ แล้วเด้งกลับมาที่หน้านี้อัตโนมัติ
จากนั้นกด "ซิงค์ข้อมูลจาก Strava เดี๋ยวนี้" เพื่อดึงรองเท้าและกิจกรรมวิ่งเข้ามา

## ข้อจำกัดที่ควรรู้

- ข้อมูลเก็บเฉพาะในเบราว์เซอร์เครื่อง/โปรไฟล์ที่ใช้งาน — ล้างแคชหรือเปลี่ยนเครื่องแล้วข้อมูลหาย เว้นแต่จะกด "ส่งออกข้อมูล (JSON)" สำรองไว้ก่อน
- การซิงค์ Strava ดึงกิจกรรมล่าสุดสูงสุด 100 รายการต่อการซิงค์หนึ่งครั้ง
- ยังไม่มีการ sync ข้อมูลระหว่างเว็บกับแอพ iOS (`ios-app/`) — เป็นข้อมูลคนละชุดกัน
