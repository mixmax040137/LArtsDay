# แบบทดสอบสื่ออินโฟกราฟิก A/B — L'Arts Day 2569

โปรแกรมแบบทดสอบออนไลน์สำหรับงานวิจัย *“การวัดผลความพึงพอใจในการปรับเปลี่ยนรูปแบบสื่ออินโฟกราฟิก”*
คณะศิลปศาสตร์ มหาวิทยาลัยธรรมศาสตร์ — เก็บข้อมูลลง Google Sheet เพื่อนำไปวิเคราะห์ต่อใน SPSS

## โครงสร้างแบบทดสอบ
1. **ข้อตกลง (Informed Consent)** — ต้องติ๊กยินยอมครบทุกข้อจึงจะเริ่มได้
2. **ส่วนที่ 1 — ข้อมูลทั่วไป** 5 ข้อ (เพศ อายุ ประเภทผู้ใช้ ความถี่ ช่องทาง)
3. **ส่วนที่ 2 — รับชมสื่อ 15 วินาที** (นาฬิกานับถอยหลัง กดข้ามไม่ได้) แล้วทำ **แบบทดสอบความเข้าใจ 5 ข้อ** ทีละข้อ จับเวลาแต่ละข้ออัตโนมัติ ย้อนกลับไม่ได้
4. **ส่วนที่ 3 — ความพึงพอใจ 19 ข้อ / 6 ด้าน** (Likert 5 ระดับ)

**2 ลิงก์แยกกันชัดเจน:**
| กลุ่ม | ลิงก์ | สื่อที่เห็น |
|---|---|---|
| **กลุ่ม A** | `group-a.html` (หรือ `index.html?group=A`) | สื่อแบบเดิม |
| **กลุ่ม B** | `group-b.html` (หรือ `index.html?group=B`) | สื่อแบบใหม่ |

---

## การติดตั้ง (ทำครั้งเดียว ~10 นาที)

### 1) เตรียม Google Sheet + Apps Script (ตัวเก็บข้อมูล)
1. สร้าง Google Sheet ใหม่ (เช่นชื่อ *LArtsDay Responses*)
2. เมนู **Extensions → Apps Script**
3. ลบโค้ดเดิมทั้งหมด แล้ววางเนื้อหาจากไฟล์ [`apps-script/Code.gs`](apps-script/Code.gs)
4. กด **Deploy → New deployment**
   - เลือกชนิด (เฟือง ⚙) → **Web app**
   - **Execute as:** Me
   - **Who has access:** **Anyone**  ← สำคัญ
5. กด Deploy แล้ว **คัดลอก “Web app URL”** (ลงท้ายด้วย `/exec`)
6. ทดสอบ: เปิด URL นั้นในเบราว์เซอร์ ควรเห็นข้อความ `{"status":"ok",...}`

### 2) ตั้งค่าในโปรแกรม
เปิดไฟล์ [`index.html`](index.html) แก้ส่วน `CONFIG` ด้านบน:
```js
const CONFIG = {
  ENDPOINT: "วาง Web app URL ที่คัดลอกมาตรงนี้",
  IMAGE_A: "images/group-a.jpg",
  IMAGE_B: "images/group-b.jpg",
  VIEW_SECONDS: 15
};
```

### 3) ใส่รูปสื่อ
วางไฟล์รูป 2 ไฟล์ในโฟลเดอร์ `images/`
- `group-a.jpg` = สื่อแบบเดิม
- `group-b.jpg` = สื่อแบบใหม่

(ดูรายละเอียดใน [`images/README.txt`](images/README.txt))

### 4) นำขึ้นออนไลน์ (เลือกวิธีใดก็ได้)
ต้อง “โฮสต์” ไฟล์เพื่อให้ได้ลิงก์แชร์ — แนะนำวิธีฟรีและง่าย:

- **GitHub Pages**: สร้าง repo → อัปโหลดทั้งโฟลเดอร์ → Settings → Pages → เลือก branch
  ได้ลิงก์ เช่น `https://ชื่อผู้ใช้.github.io/repo/group-a.html`
- **Netlify Drop**: ลากโฟลเดอร์ทั้งอันไปวางที่ https://app.netlify.com/drop (ได้ลิงก์ทันที)
- **Google Sites**: ฝังหน้าเว็บผ่าน Embed (ต้องโฮสต์ไฟล์ก่อน)

> เปิดไฟล์แบบดับเบิลคลิก (file://) ก็ทำงานได้ แต่จะ **แชร์ลิงก์ให้คนอื่นไม่ได้** และบางเบราว์เซอร์
> อาจบล็อกการส่งข้อมูล จึงควรโฮสต์ออนไลน์เมื่อใช้เก็บข้อมูลจริง

### 5) แจกลิงก์
- ส่งลิงก์ **group-a.html** ให้ผู้ตอบกลุ่ม A
- ส่งลิงก์ **group-b.html** ให้ผู้ตอบกลุ่ม B

> การสุ่มแบ่งกลุ่ม (Systematic Random Assignment) ทำได้โดยสลับแจก A/B ทีละคน
> หรือใช้รายชื่อเลขคู่ = A, เลขคี่ = B ตามที่ระบุในบทที่ 3

## การดึงข้อมูลไปวิเคราะห์ SPSS
1. เปิด Google Sheet → ชีต `responses`
2. **File → Download → Microsoft Excel (.xlsx)** หรือ **.csv**
3. ใน SPSS: **File → Import Data → Excel/CSV** (ติ๊กใช้แถวแรกเป็นชื่อตัวแปร)
4. ดูความหมายและการลงรหัสตัวแปรทั้งหมดได้ที่ [`spss/codebook.md`](spss/codebook.md)

---

## ทดสอบก่อนใช้จริง
- เปิด `group-a.html` และ `group-b.html` ลองทำจนจบ 1 รอบ
- ตรวจว่ามีข้อมูล 1 แถวเพิ่มในชีต `responses` และ `group` ตรงกับลิงก์ที่เปิด
- ลบแถวทดสอบออกก่อนเก็บข้อมูลจริง (เก็บหัวตารางแถวแรกไว้)

## หมายเหตุทางเทคนิค
- ทำงานได้ทุกอุปกรณ์/แพลตฟอร์ม (มือถือ แท็บเล็ต คอมพิวเตอร์) ทุกเบราว์เซอร์สมัยใหม่
- ไม่เก็บชื่อหรือข้อมูลที่ระบุตัวตน — มีเพียงรหัสสุ่ม `respondent_id`
- จับเวลาด้วย `performance.now()` แม่นยำระดับมิลลิวินาที
- ถ้าส่งข้อมูลไม่สำเร็จ ระบบให้กด “ลองอีกครั้ง” หรือคัดลอกข้อมูลส่งผู้วิจัยทางอีเมลได้
